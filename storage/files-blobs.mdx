---
title: "Upload, download, and manage files in Clorabase"
description: "Learn how to upload regular files and large blobs, download file contents, list directory entries, and delete files from Clorabase Storage."
---

This page covers every storage operation available in the Clorabase SDK. For a conceptual overview of the two storage tiers, see [Clorabase Storage overview](/storage/overview).

## Navigate directories

All operations start from a `ClorabaseStorage` reference. Use `directory()` to scope a reference to a subdirectory. The directory is created automatically when the first file is written into it.

<Tabs>
  <Tab title="Android">
    ```java
    ClorabaseStorage storage = clorabase.getStorage();
    ClorabaseStorage images  = storage.directory("images");
    ClorabaseStorage avatars = storage.directory("images").directory("avatars");
    ```
  </Tab>
  <Tab title="Java">
    ```java
    ClorabaseStorage storage = clorabase.getStorage();
    ClorabaseStorage images  = storage.directory("images");
    ClorabaseStorage avatars = storage.directory("images").directory("avatars");
    ```
  </Tab>
</Tabs>

## Upload a regular file

### Small files — `addFile()`

Use `addFile()` when the content is already in memory as a byte array. This is synchronous in the Java SDK and returns a `Task<Void>` in Android.

<Warning>
GitHub encodes committed content as base64, which inflates size by ~33 %. Keep `addFile()` usage under ~1 MB to stay well within GitHub's content API limits.
</Warning>

<Tabs>
  <Tab title="Android">
    ```java
    byte[] data = "Hello, Clorabase!".getBytes();

    storage.addFile(data, "hello.txt")
        .addOnSuccessListener(unused -> Log.d("Storage", "File added"))
        .addOnFailureListener(e -> Log.e("Storage", "Upload failed", e));
    ```
  </Tab>
  <Tab title="Java">
    ```java
    byte[] data = "Hello, Clorabase!".getBytes();

    try {
        storage.addFile(data, "hello.txt");
    } catch (StorageException e) {
        System.err.println("Upload failed: " + e.getMessage());
    }
    ```
  </Tab>
</Tabs>

### Larger files — `uploadFile()`

Use `uploadFile()` to stream a file from an `InputStream` with live progress callbacks. The file must be under 50 MB.

<Note>
The Android `ProgressListener.onComplete()` takes no parameters. The Java `ProgressListener.onComplete(String result)` receives the raw API response string from GitHub.
</Note>

<Tabs>
  <Tab title="Android">
    ```java
    InputStream fileStream = getContentResolver().openInputStream(fileUri);

    storage.uploadFile(fileStream, "photo.jpg", new ProgressListener() {
        @Override
        public void onProgress(long bytesRead, long totalBytes) {
            int percent = (int) (bytesRead * 100 / totalBytes);
            progressBar.setProgress(percent);
        }

        @Override
        public void onComplete() {
            Toast.makeText(context, "Upload complete", Toast.LENGTH_SHORT).show();
        }

        @Override
        public void onError(Exception e) {
            Log.e("Storage", "Upload failed", e);
        }
    });
    ```
  </Tab>
  <Tab title="Java">
    ```java
    InputStream fileStream = new FileInputStream("photo.jpg");

    storage.uploadFile(fileStream, "photo.jpg", new ProgressListener() {
        @Override
        public void onProgress(long bytesRead, long totalBytes) {
            System.out.printf("Progress: %d / %d bytes%n", bytesRead, totalBytes);
        }

        @Override
        public void onComplete(String result) {
            System.out.println("Upload complete. Response: " + result);
        }

        @Override
        public void onError(Exception e) {
            System.err.println("Upload failed: " + e.getMessage());
        }
    });
    ```
  </Tab>
</Tabs>

<Warning>
Passing a stream whose available bytes exceed 50 MB throws a `StorageException` with reason `FILE_TOO_LARGE`. Use `uploadBlob()` instead for large files.
</Warning>

## Upload a blob (large file)

`uploadBlob()` uploads the file as a GitHub Release asset, then writes a `.ptr` pointer file into the repo directory. This consumes two GitHub API calls per upload.

<Info>
A GitHub Release must exist for your project before calling `uploadBlob()`. See the [storage overview](/storage/overview) for setup instructions.
</Info>

<Tabs>
  <Tab title="Android">
    ```java
    InputStream videoStream = getContentResolver().openInputStream(videoUri);

    storage.uploadBlob(videoStream, "intro.mp4", new ProgressListener() {
        @Override
        public void onProgress(long bytesRead, long totalBytes) {
            int percent = (int) (bytesRead * 100 / totalBytes);
            progressBar.setProgress(percent);
        }

        @Override
        public void onComplete() {
            Toast.makeText(context, "Blob uploaded", Toast.LENGTH_SHORT).show();
        }

        @Override
        public void onError(Exception e) {
            Log.e("Storage", "Blob upload failed", e);
        }
    });
    ```
  </Tab>
  <Tab title="Java">
    ```java
    InputStream videoStream = new FileInputStream("intro.mp4");

    storage.uploadBlob(videoStream, "intro.mp4", new ProgressListener() {
        @Override
        public void onProgress(long bytesRead, long totalBytes) {
            System.out.printf("Progress: %d / %d bytes%n", bytesRead, totalBytes);
        }

        @Override
        public void onComplete(String result) {
            // result is the browser_download_url of the uploaded asset
            System.out.println("Blob available at: " + result);
        }

        @Override
        public void onError(Exception e) {
            System.err.println("Blob upload failed: " + e.getMessage());
        }
    });
    ```
  </Tab>
</Tabs>

<Note>
Clorabase automatically appends `.ptr` to the `fileName` argument when creating the pointer file. You do not need to include it when calling `uploadBlob()`.
</Note>

## Download a regular file

`getFile()` returns the file contents as a `byte[]`. In Android it returns `Task<byte[]>` and runs on a background thread.

<Tabs>
  <Tab title="Android">
    ```java
    storage.getFile("hello.txt")
        .addOnSuccessListener(bytes -> {
            String content = new String(bytes);
            textView.setText(content);
        })
        .addOnFailureListener(e -> Log.e("Storage", "Download failed", e));
    ```
  </Tab>
  <Tab title="Java">
    ```java
    try {
        byte[] bytes = storage.getFile("hello.txt");
        String content = new String(bytes);
        System.out.println(content);
    } catch (StorageException e) {
        System.err.println("Download failed: " + e.getMessage());
    }
    ```
  </Tab>
</Tabs>

## Download a blob

Two methods are available for blobs. Both require the `.ptr` file name (including the `.ptr` extension).

### Stream the blob — `getBlobStream()`

Returns an `InputStream` connected directly to the GitHub Release asset. In Android it returns `Task<InputStream>`.

<Tabs>
  <Tab title="Android">
    ```java
    storage.getBlobStream("intro.mp4.ptr")
        .addOnSuccessListener(inputStream -> {
            // pipe inputStream to a file or media player
        })
        .addOnFailureListener(e -> Log.e("Storage", "Stream failed", e));
    ```
  </Tab>
  <Tab title="Java">
    ```java
    try {
        InputStream stream = storage.getBlobStream("intro.mp4.ptr");
        // pipe stream to a file or media player
    } catch (StorageException e) {
        System.err.println("Stream failed: " + e.getMessage());
    }
    ```
  </Tab>
</Tabs>

### Get the download URL — `getBlobDownloadURL()`

Returns a `java.net.URL` pointing directly at the Release asset. Useful when you need to hand a URL to a media player or download manager rather than reading the bytes yourself.

<Note>
`getBlobDownloadURL()` is available in the Java SDK only. The Android SDK exposes `getBlobStream()` for async access.
</Note>

```java
try {
    URL url = storage.getBlobDownloadURL("intro.mp4.ptr");
    System.out.println("Download at: " + url);
} catch (StorageException e) {
    System.err.println("Failed: " + e.getMessage());
}
```

<Warning>
Passing a file name that does not end in `.ptr` to either `getBlobStream()` or `getBlobDownloadURL()` throws an `IllegalArgumentException` immediately, before any network call is made.
</Warning>

## List files

`listFiles()` returns the names of all files and subdirectories in the current directory reference.

<Tabs>
  <Tab title="Android">
    ```java
    storage.listFiles()
        .addOnSuccessListener(names -> {
            for (String name : names) {
                Log.d("Storage", name);
            }
        })
        .addOnFailureListener(e -> Log.e("Storage", "List failed", e));
    ```
  </Tab>
  <Tab title="Java">
    ```java
    try {
        List<String> names = storage.listFiles();
        names.forEach(System.out::println);
    } catch (StorageException e) {
        System.err.println("List failed: " + e.getMessage());
    }
    ```
  </Tab>
</Tabs>

## Delete files

### Delete a regular file — `delete()`

<Tabs>
  <Tab title="Android">
    ```java
    storage.delete("hello.txt")
        .addOnSuccessListener(unused -> Log.d("Storage", "Deleted"))
        .addOnFailureListener(e -> Log.e("Storage", "Delete failed", e));
    ```
  </Tab>
  <Tab title="Java">
    ```java
    try {
        storage.delete("hello.txt");
    } catch (StorageException e) {
        System.err.println("Delete failed: " + e.getMessage());
    }
    ```
  </Tab>
</Tabs>

### Delete a blob — `deleteBlob()`

`deleteBlob()` removes both the GitHub Release asset and its `.ptr` pointer file. Pass the file name **without** the `.ptr` extension.

<Tabs>
  <Tab title="Android">
    ```java
    storage.deleteBlob("intro.mp4")
        .addOnSuccessListener(unused -> Log.d("Storage", "Blob deleted"))
        .addOnFailureListener(e -> Log.e("Storage", "Blob delete failed", e));
    ```
  </Tab>
  <Tab title="Java">
    ```java
    try {
        storage.deleteBlob("intro.mp4");
    } catch (StorageException e) {
        System.err.println("Blob delete failed: " + e.getMessage());
    }
    ```
  </Tab>
</Tabs>

<Warning>
`deleteBlob()` is irreversible. The Release asset is permanently removed from GitHub and cannot be recovered.
</Warning>
