Clorabase is a lightweight, serverless **Backend-as-a-Service (BaaS)** that uses GitHub as your entire backend infrastructure. Instead of setting up servers or paying for cloud services, you store app data, files, and messages directly in a private GitHub repository. It's completely free and requires only a GitHub account.

## Why Clorabase?

Clorabase is designed for developers who want a simple, zero-cost backend for small Android or Java apps — hobby projects, prototypes, or apps with a small user base.

## How it works

Clorabase stores all your app data inside a private GitHub repository called `Clorabase-projects`. Each project you create gets a folder in that repo containing:

- **`db/`** — encrypted JSON document files for your database
- **`storage/`** — binary files or pointers to GitHub Release assets
- **`messages/`** — in-app message payloads for Android
- **`updates/`** — version metadata for in-app update prompts
- **`config.json`** — project metadata

The SDK communicates directly with the GitHub API from your app — there is no intermediate server.

## Get started

<Steps>
  <Step title="Generate a GitHub token">
    Create a Personal Access Token with `repo` scope in your GitHub settings. See the [setup guide](/setup) for exact steps.
  </Step>
  <Step title="Add the dependency">
    Add the JitPack repository and the Clorabase dependency to your `build.gradle`. See the [quickstart](/quickstart).
  </Step>
  <Step title="Initialize the SDK">
    Call `Clorabase.getInstance(username, token, project)` once at app startup.
  </Step>
  <Step title="Read and write data">
    Use `getDatabase()` and `getStorage()` to start persisting data immediately.
  </Step>
</Steps>

<Note>
Clorabase is best suited for small apps, hobby projects, and non-commercial use cases. For apps requiring authentication, advanced querying, or millions of users, consider Firebase or Supabase.
</Note>
